# 项目文档目录说明

本目录用于统一管理项目需求、方案设计、执行记录、版本节点和回退依据。

## 目录结构

- `00-governance/`：文档治理规则、维护制度、评审要求、命名规范。
- `01-requirements/`：需求文档，包括需求背景、目标、范围、验收标准、变更记录。
- `02-design/`：实现方案设计，包括架构设计、模块设计、技术选型、风险分析。
- `03-execution/`：执行过程记录，包括任务拆解、实施日志、问题记录、测试结果。
- `04-versions/`：按时间节点保存的重要文档快照，用于里程碑回溯和进度回退依据。
- `05-changelog/`：文档级与项目级变更记录。
- `06-retrospective/`：阶段复盘、经验总结、后续优化建议。
- `_templates/`：文档模板和复用骨架。
- `archive/`：归档文档，保留历史但不再持续更新。
- `local/`：本地工作草稿、个人笔记、临时材料，已在 `.gitignore` 中忽略。
- `exports/`：导出文件，如 PDF、截图包、交付压缩包，已在 `.gitignore` 中忽略。
- `tmp/`：临时整理文件，已在 `.gitignore` 中忽略。

## 使用原则

1. 正式项目文档必须放在 `document/` 下对应分类目录。
2. 所有正式文档应使用统一命名规则，并在文档头部声明版本、状态、负责人、更新时间。
3. 需求、设计、执行、版本快照之间必须建立可追溯关系。
4. 里程碑节点必须在 `04-versions/` 中留存快照，作为回退和审计依据。
5. 临时材料不得替代正式文档；需要留档时，应整理后转入正式目录。
